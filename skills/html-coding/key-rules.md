# [html-coding] HTML コーディング規約サマリー

## 構造・セマンティクス

- 要素は意味に合ったタグを使う（`<div>` の乱用禁止）
- 見出しは階層順に使う（`h1` → `h2` → `h3`、スキップしない）
- ページ内に `<h1>` は1つだけ
- リストは `<ul>` / `<ol>` / `<dl>` を用途に応じて使い分ける
- テーブルはデータ用途のみ（レイアウト目的に使わない）
- `<button>` はアクション用、`<a>` は遷移用（役割を混同しない）

## 属性

- すべての `<img>` に `alt` 属性を書く（装飾画像は `alt=""`）
- `<a>` の `href` に `#` のみ使わない（意味のあるURLまたは `button` を使う）
- `<input>` には必ず `<label>` を紐付ける（`for` + `id` または `aria-label`）
- `target="_blank"` には必ず `rel="noopener noreferrer"` を追加する
- boolean属性は値を省略する（`disabled` ◯ / `disabled="disabled"` ✗）

## フォーム

- `<form>` には `action` と `method` を明示する
- 必須項目には `required` 属性と視覚的なマーク（`*` など）を付ける
- エラーメッセージは該当フィールドに `aria-describedby` で紐付ける

## よくある間違い

- インラインスタイル（`style="..."`）をHTMLに書く → CSS に移す
- `<br>` を段落の代わりに使う → `<p>` を使う
- クリックイベントを `<div>` に付ける → `<button>` を使う
- 空の `alt` を省略する → `alt=""` と明示する（スクリーンリーダー対策）
